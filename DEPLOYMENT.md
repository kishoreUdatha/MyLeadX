## Option 1: Railway (Recommended for Quick Start)

### Step 1: Create Railway Account
1. Go to https://railway.app
2. Sign up with GitHub

### Step 2: Deploy Backend
```bash
# In backend folder
railway login
railway init
railway add --database postgresql
railway up
```

### Step 3: Deploy Frontend
```bash
# In frontend folder
railway init
railway up
```

### Step 4: Set Environment Variables
In Railway Dashboard, add these variables:

```env
# Database (auto-generated by Railway)
DATABASE_URL=postgresql://...

# JWT
JWT_SECRET=your-super-secret-jwt-key-min-32-chars
JWT_REFRESH_SECRET=your-refresh-secret-key-min-32-chars

# Server
NODE_ENV=production
PORT=3000

# OpenAI
OPENAI_API_KEY=sk-...

# Voice Provider (choose one)
VOICE_PROVIDER=exotel
EXOTEL_ACCOUNT_SID=...
EXOTEL_API_KEY=...
EXOTEL_API_TOKEN=...
EXOTEL_CALLER_ID=...

# Razorpay (for payments)
RAZORPAY_KEY_ID=...
RAZORPAY_KEY_SECRET=...

# Super Admin
SUPER_ADMIN_SETUP_KEY=your-secret-setup-key
```

## Option 2: Render

### Step 1: Create render.yaml
Create `render.yaml` in root:

```yaml
services:
  # Backend API
  - type: web
    name: crm-backend
    env: node
    buildCommand: cd backend && npm install && npx prisma generate
    startCommand: cd backend && npm start
    envVars:
      - key: NODE_ENV
        value: production
      - key: DATABASE_URL
        fromDatabase:
          name: crm-db
          property: connectionString

  # Frontend
  - type: web
    name: crm-frontend
    env: static
    buildCommand: cd frontend && npm install && npm run build
    staticPublishPath: frontend/dist
    envVars:
      - key: VITE_API_URL
        value: https://crm-backend.onrender.com

databases:
  - name: crm-db
    plan: starter
```

### Step 2: Deploy
1. Go to https://render.com
2. Connect GitHub repo
3. Click "New Blueprint"
4. Select your repo

## Option 3: Docker Deployment

### Dockerfile for Backend

Create `backend/Dockerfile`:

```dockerfile
FROM node:18-alpine

WORKDIR /app

# Install dependencies
COPY package*.json ./
RUN npm ci --only=production

# Copy source
COPY . .

# Generate Prisma client
RUN npx prisma generate

# Build
RUN npm run build

EXPOSE 3000

CMD ["npm", "start"]
```

### Dockerfile for Frontend

Create `frontend/Dockerfile`:

```dockerfile
FROM node:18-alpine as builder

WORKDIR /app

COPY package*.json ./
RUN npm ci

COPY . .
RUN npm run build

# Production
FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80
```

### docker-compose.yml

Create `docker-compose.yml` in root:

```yaml
version: '3.8'

services:
  postgres:
    image: postgres:15
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres123
      POSTGRES_DB: crm_lead_generation
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

  backend:
    build: ./backend
    ports:
      - "3000:3000"
    environment:
      - DATABASE_URL=postgresql://postgres:postgres123@postgres:5432/crm_lead_generation
      - REDIS_URL=redis://redis:6379
      - NODE_ENV=production
    depends_on:
      - postgres
      - redis

  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend

volumes:
  postgres_data:
```

### Deploy with Docker

```bash
# Build and run
docker-compose up -d

# View logs
docker-compose logs -f

# Stop
docker-compose down
```

## Option 4: AWS Deployment

### Architecture

```
┌─────────────────────────────────────────────────────────┐
│                        AWS                              │
├─────────────────────────────────────────────────────────┤
│  CloudFront (CDN) → S3 (Frontend)                      │
│         ↓                                               │
│  ALB (Load Balancer)                                    │
│         ↓                                               │
│  ECS/Fargate (Backend containers)                       │
│         ↓                                               │
│  RDS PostgreSQL + ElastiCache Redis                     │
└─────────────────────────────────────────────────────────┘
```

### Step 1: Create RDS Database
- Go to AWS RDS
- Create PostgreSQL instance
- Note the connection string

### Step 2: Create ECR Repository
```bash
aws ecr create-repository --repository-name crm-backend
aws ecr create-repository --repository-name crm-frontend
```

### Step 3: Push Docker Images
```bash
# Login to ECR
aws ecr get-login-password | docker login --username AWS --password-stdin YOUR_ACCOUNT_ID.dkr.ecr.REGION.amazonaws.com

# Build and push
docker build -t crm-backend ./backend
docker tag crm-backend:latest YOUR_ACCOUNT_ID.dkr.ecr.REGION.amazonaws.com/crm-backend:latest
docker push YOUR_ACCOUNT_ID.dkr.ecr.REGION.amazonaws.com/crm-backend:latest
```

### Step 4: Create ECS Service
- Create ECS cluster
- Create task definition with your images
- Create service with ALB

## Environment Variables Reference

### Required Variables

```env

DATABASE_URL=postgresql://user:pass@host:5432/dbname

# JWT (generate secure random strings)
JWT_SECRET=min-32-character-random-string
JWT_REFRESH_SECRET=another-32-character-random-string

NODE_ENV=production
PORT=3000

# Frontend URL (for CORS)

BASE_URL=https://your-backend-domain.com

OPENAI_API_KEY=sk-...
```

## Scaling Recommendations

### For 100-500 users:
- Railway/Render (starter plan)
- Single instance
- Cost: $10-30/month

### For 500-5000 users:
- Multiple backend instances
- Load balancer
- Redis for sessions
- Cost: $50-150/month

### For 5000+ users:
- AWS/GCP with auto-scaling
- Kubernetes/ECS
- CDN for frontend
- Read replicas for database
- Cost: $200-1000/month
